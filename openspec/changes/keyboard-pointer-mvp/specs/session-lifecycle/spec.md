## ADDED Requirements

### Requirement: グローバルショートカットによる起動
本ツールはWindows上のどのアプリがフォアグラウンドであっても、グローバルショートカットで選択セッションを開始できなければならない（SHALL）。

#### Scenario: 任意のアプリから起動できる
- **WHEN** 任意のアプリがフォアグラウンドの状態で起動ホットキー（既定 `Alt+Space`）を押す
- **THEN** 対象モニターにオーバーレイを表示し、選択セッションを開始する

#### Scenario: ホットキー登録に失敗した場合
- **WHEN** 起動時にホットキー登録に失敗する
- **THEN** 設定画面を開き、ホットキー変更を促す

### Requirement: 多重起動の防止
選択セッションが進行中の間、本ツールは新たなセッションを多重に開始してはならない（MUST NOT）。

#### Scenario: セッション中の再押下を無視する
- **WHEN** 選択セッションが進行中に起動ホットキーが再度押される
- **THEN** 新しいセッションを開始せず、進行中のセッションを維持する

### Requirement: モニターキャプチャの取得と破棄
セッション開始時に対象モニターの物理ピクセル画像をメモリ上に取得し（SHALL）、ファイルへ保存してはならず（MUST NOT）、セッション終了時に破棄しなければならない（SHALL）。

#### Scenario: 開始時にキャプチャする
- **WHEN** セッションが開始される
- **THEN** 対象モニターの物理ピクセル画像をメモリ上に取得する

#### Scenario: キャプチャをファイル保存しない
- **WHEN** セッションがキャプチャ画像を保持している
- **THEN** その画像をディスク上のファイルへ書き出さない

#### Scenario: 終了時に破棄する
- **WHEN** セッションが終了する（クリック実行・キャンセル・エラーのいずれでも）
- **THEN** 保持していたキャプチャ画像を破棄する

#### Scenario: キャプチャに失敗した場合
- **WHEN** セッション開始時に画面キャプチャに失敗する
- **THEN** ユーザーに通知してセッションを開始せず終了する

### Requirement: 状態遷移
セッションは Idle → Capturing → Selecting → (Zooming → Selecting)* → Executing/Cancelled → Idle の状態遷移に従わなければならない（SHALL）。

#### Scenario: 通常ラベルで絞り込む遷移
- **WHEN** Selecting 状態で通常ラベルが入力される
- **THEN** Zooming を経て再び Selecting 状態となる

#### Scenario: Shift付きラベルでクリック実行へ遷移
- **WHEN** Selecting 状態で Shift付きラベルが入力される
- **THEN** Executing 状態へ遷移し、クリック実行後に Idle へ戻る

### Requirement: エラー時の安全な復帰
内部エラーやパニックが発生した場合、本ツールはオーバーレイを閉じて通常操作へ戻らなければならない（SHALL）。ログにキャプチャ画像・入力ラベル列・クリック座標を含めてはならない（MUST NOT）。

#### Scenario: 内部エラーで通常状態へ戻る
- **WHEN** セッション中に内部エラーが発生する
- **THEN** オーバーレイを閉じ、Idle 状態へ戻す

#### Scenario: パニック時にログを保存する
- **WHEN** 予期しないパニックが発生する
- **THEN** オーバーレイを閉じ、キャプチャ画像・ラベル列・座標を含まないログを保存する
