## ADDED Requirements

### Requirement: 設定ファイルの読み書き
本ツールは設定を `%APPDATA%\<ProjectName>\config.json` に保存し、起動時に読み込まなければならない（SHALL）。設定項目は起動ホットキー（左/右/ダブル）、ラベル数、最大段階数、ラベルサイズ、アニメーション有無、Windows自動起動を含む。

#### Scenario: 設定を保存して再読み込みする
- **WHEN** GUIで設定を変更して保存する
- **THEN** `config.json` に書き出し、次回起動時にその値を読み込む

### Requirement: 初期設定なしでの即利用
本ツールはインストール直後、初期設定なしで既定値により利用できなければならない（SHALL）。

#### Scenario: 設定ファイルがなくても起動する
- **WHEN** `config.json` が存在しない状態で起動する
- **THEN** 既定値（`Alt+Space`/`Alt+R`/`Alt+D`、ラベル数25、最大段階5、ラベル標準、アニメーション有効、自動起動有効）で動作する

### Requirement: GUIによる設定変更
本ツールはGUIから主要設定を変更できなければならない（SHALL）。提供する項目は、左/右/ダブルクリック起動キー、ラベルサイズ、ラベル数、最大段階数、アニメーション有無、Windows自動起動、チュートリアル再表示、設定初期化とする。

#### Scenario: GUIで設定項目を変更する
- **WHEN** ユーザーが設定画面で項目を変更する
- **THEN** 変更が即時または保存時に反映され、`config.json` に永続化される

#### Scenario: 設定を初期化する
- **WHEN** ユーザーが設定初期化を実行する
- **THEN** すべての設定を既定値に戻す

### Requirement: 破損した設定ファイルの扱い
設定ファイルが破損していた場合、本ツールは既定値で起動し、破損ファイルを退避しなければならない（SHALL）。

#### Scenario: 破損時に既定値で起動する
- **WHEN** `config.json` が破損していて読み込めない
- **THEN** 既定値で起動し、破損ファイルを別名へ退避する
